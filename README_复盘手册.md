# 🧪 离子液体-制冷剂 Tri-Graph GNN 预测项目复盘手册

本手册是为了记录从“数据输入”到“神经网络预测”全流程搭建逻辑的复盘。主要帮助后续针对不同考点（如：扩散系数预测、不同制冷剂）进行快速迁移和除错。

---

## 🗺️ 核心架构全景图（数据流向）

整个项目的流向遵循绝对的流水线法则：
1. **[原始文本]** $\rightarrow$ 2. **[SMILES图纸]** $\rightarrow$ 3. **[RDKit 化学推演矩阵]** $\rightarrow$ 4. **[图神经网络(GNN)]** $\rightarrow$ 5. **[最终物理属性]**

## 📂 脚本功能模块剖析

### 1️⃣ 数据前处理工厂：`prepare_tri_graph_data.py`
**【当前状态】**：完全吃透并已修复（加装了 30 多种制冷剂以及遗漏的稀有离子液体）。
**【核心逻辑】**：
这是一个“翻译与数字建模”的枢纽。你的总结完全正确，其内部运行齿轮如下：

1. **查户口（SMILES 匹配）**：
   - 模型是不认识字母或 Excel 名称的。脚本会拿 Excel 中的简写（如 `R32`、`[BMIM]`），去咱们建立的花名册大字典（`smiles_dict` 连同手动补丁 `extra_smiles`）里，对齐查出对应的全球唯一 SMILES 化学代码（如 `C(F)F`）。
   - **坑点**：如果缺任何一个，这行实验数据就会被废弃。（我们已通过探测脚本 `find_missing_smiles.py` 把丢弃的 1600 多条数据全部拯救回来了）。
2. **算特征（RDKit 物理体检）**：
   - 调用底层的 `rdkit.Chem`，把这串一维的 SMILES 代码直接复原成包含氢键、双键等信息的立体模型分子。
   - 分别通过 `get_atom_features`（检测原子序数、杂化、芳香性、连键数、电荷）和 `get_bond_features`（检测单双键类型、芳香环属性）将化学特征转化为**全数字矩阵**。
3. **出货（组装图结构大盲盒）**：
   - 提取出原子的特征矩阵 `node_f`、连边的地图手册 `edge_index`、边属性 `edge_attr`。
   - 把 [阳图，阴图，制冷图，温度 T，压力 P] 装进一个记录单元，并将最终结果存为 `data.npy` 留给下一步调用。

**【未来如何修改】**：
- 如果加了新离子或新制冷剂 $\rightarrow$ **手动补全字典 `extra_smiles`**。
- 如果不预测 x1 (溶解度) 改预测扩散系数 $\rightarrow$ **修改读取 Excel 的 sheet_name，并把目标标签换成对应的新列头。**

### 2️⃣ 图结构微操与“全局节点”注入：`Dataset.py`
**【当前状态】**：核心逻辑已吃透，并修复了单原子离子（如 `[Cl-]`）无特征边导致的 Pytorch 崩溃 Bug。
**【核心逻辑】**：
刚才从 `data.npy` 读取出来的数据还只是毫无感情的 Python 嵌套列表。这个脚本它是 **图神经网络（GNN）的翻译官与拼图大师**。

1. **化身为 PyTorch 张量（`mol2graph` 函数）**：
   - 真正把数字列表转化为了能够在高端显卡（GPU）中做超高速并行计算的高维张量（`torch.LongTensor`）。
   - 让它正式挂牌，成为具有 `x` (特征阵), `edge_index` (边地图), `edge_attr` (边属性阵) 的标准 `torch_geometric.data.Data` 图对象。
2. **三图合一（`combine_Graph` 函数）**：
   - 使用 PyG 框架的 `Batch.from_data_list([cation, anion, refri])` 将三种本毫无瓜葛的独立分子在结构空间上拼成一张**“巨型孤岛网络”（Tri-Graph）**。
3. **点睛之笔：植入上帝视角的全局节点（`add_global` 函数）**：
   - 这是你导师最关心的**模型灵魂核心（也事关后面的可解释性 Explainable 机制）**！
   - **创建节点**：代码在此凭空生成了一个额外的点（特征记为 `[0,0,0,0,0]`），并拼接到这幅巨图中，我们叫它“全局监控者”。
   - **强势打通连线**：给这个黑客点牵出成百灵活根线，**让它和阳离子、阴离子、制冷剂中所有的普通原子都全部发生双向直连**（`bi_direction=True`）。
   - **物理意义**：在真实物理世界里，你不可能指望阳离子的碳原子和制冷剂发生化学键互连。但在这套算法中，这个**全局节点充当了三者的“信息传声筒”**！随着模型层层的卷积，所有分子的独立特征首先被汇总给“监控者”，监控者再把融合后的整体特性反哺分配给所有的原子。这就是为什么在彼此不相连的情况下，神经网络依然能抽象学习到“这种阴阳离子配比”面对“这种制冷剂”为什么会表现出如此独特溶解度背后的**微观交互机理**。

**【未来如何修改】**：
- 如果要引入“第四种混合物”（如加了共溶剂），不仅要在预处理算特征，还要在 `combine_Graph` 的大括号里加上它。
- 如果想做对照组切断分子的信息交互来验证论文价值：修改顶部的开关参数 `args['add_global'] = False` 即可（注：一旦截断全局节点，由于三张图互相不连通，溶解度预测的精度大概率会立刻雪崩断崖式下跌，但这正是证明你们模型架构优越性的绝佳对比点）。

### 3️⃣ GNN 模型真正的“大脑”：`Model.py`
**【当前状态】**：已解析全流程，并进行了设备兼容性修护（移除硬编码的 `.cuda()`）。
**【核心逻辑】**：
模型使用的是 **GINEConv** (带有边特征的图同构网络)，这是目前处理小分子图最顶级的算法之一。

1. **多重特征嵌入 (Embedding)**：
   - 针对 5 个原子特征分别建立 Embedding 层。它能把离散的数字（如：杂化类型 3）映射到一个 128 维度的连续空间，让神经网络能学习到“sp3杂化”背后的电子云密度含义。
2. **多层信息传播**：
   - 堆叠了 5 层卷积层。每一层卷积，原子都会向邻居打听一次消息。
   - **全局节点的吸星大法**：由于全局节点和所有原子相连，第一遍卷积后，它就拿到了所有原子的初级信息；到第五层卷积时，它已经通过间接传递，深刻理解了整个混合体系的非共价相互作用。
3. **特征抓取 (Extract Logic)**：
   - 模型在 `forward` 结束时，会根据 `batch` 信息精准地只切出**每一个样本图中的最后一个节点特征**。
   - **这就是全局节点！** 它是这个混合系统的“汇报官”。
4. **决策头 (Pred Head)**：
   - 给全局节点的特征再拼上 [温度 T, 压力 P]。
   - 经过三层全连接网络 (MLP)，将极其复杂的化学拓扑信号压缩成一个最终数值：溶解度。

**【未来如何修改】**：
- 如果想换模型（如 GAT、GCN），只需在 Runner 里更改实例化的类名。 
- 如果想加更复杂的环境参数（如加入浓度作为输入变量），需要在这里修改 `pred_head` 的输入维度。

### 4️⃣ 指挥官控制台：`GIN_Runner.py`
**【当前状态】**：模型已跑通，实现了训练、验证、测试、三图联画。
**【核心技巧】**：
1. **余弦退火学习率 (CosineAnnealingLR)**：
   - 这里的学习率不是死板不变的，而是像波浪一样缓慢下降。这能有效防止模型掉进局部死胡同，也是 R² 能达到 0.95 的策略功臣。
2. **MAE 损失函数**：
   - 相比 MSE，MAE 对离群值更不敏感，对于这种实验测量数据（可能存在个别测得不准的数据点），MAE 能让训练更稳健。
3. **自动化模型保存**：
   - 代码每次发现验证集（Validation）变好了且没有过拟合，就会自动覆盖保存 `best_model_para.pth`。

**【操作手册】**：
- 修改 `batch_size`：如果显存不够就调小，如果想跑得快就调大。
- 观察 `GIN.png`：散点图越靠近 45 度斜线，说明预测越牛。
- **全量数据运行**：现在数据已扩充至 4444 条，直接命令行运行此脚本即可。
